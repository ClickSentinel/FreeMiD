# Security Policy

## Supported versions

The latest release is the only actively maintained version. Security fixes are applied to the current release and shipped as a new version; no backport branches are maintained.

## Reporting a vulnerability

If you believe you have found a security issue in FreeMiD, **do not open a public issue**.

Report it privately by emailing [security@freemid.ca](mailto:security@freemid.ca), or use [GitHub's private vulnerability reporting](https://github.com/ClickSentinel/FreeMiD/security/advisories/new).

Include:

- A short description of the issue
- Steps to reproduce
- Any affected versions
- Any logs, screenshots, or proof of concept you can safely share

## What we will do

We will review reports as soon as practical and respond with next steps. If the issue is confirmed, we will coordinate a fix and disclose publicly after a patch is released.
